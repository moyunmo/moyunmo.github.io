把ReactNative的图片加载框架替换成SDWebImage

刚接触RN的时候，就想接入RN的话图片的缓存逻辑要维护两套并且缓存不能共享， 心里嘀咕了一下，能不能替换成native端的SDWebImage去下载、缓存。得益于RN强大的组件化能力以及良好的封装，仅仅加几行代码就解决了。

之前看RN图片加载的源码分析,官方提供的下载仅仅是通过NSURLSession下载，缓存也是NSURLRequestCache缓存,RN的加载远程图片的核心逻辑在RCTImageLoader._loadImageOrDataWithURLRequest，首先会找到合适的图片加载器

// Find suitable image URL loader
id<RCTImageURLLoader> loadHandler = [self imageURLLoaderForURL:request.URL];

内部会去在RN初始化的所有moudle里面去找实现’RCTImageURLLoader’的类， 这里新建一个类 CMImageLoader，实现RCTImageURLLoader,RCTBridgeModule协议

@interface CMImageLoader : NSObject<RCTImageURLLoader,RCTBridgeModule>

@end

并在.m文件实现对应的协议方法

RCT_EXPORT_MODULE();

- (BOOL)canLoadImageURL:(NSURL *)requestURL
{
    return YES;
}

- (RCTImageLoaderCancellationBlock)loadImageForURL:(NSURL *)imageURL
                                              size:(CGSize)size
                                             scale:(CGFloat)scale
                                        resizeMode:(RCTResizeMode)resizeMode
                                   progressHandler:(RCTImageLoaderProgressBlock)progressHandler
                                partialLoadHandler:(RCTImageLoaderPartialLoadBlock)partialLoadHandler
                                 completionHandler:(RCTImageLoaderCompletionBlock)completionHandler
{
    __block atomic_bool cancelled = ATOMIC_VAR_INIT(NO);
    if (RCTIsLocalAssetURL(imageURL)) {
        RCTExecuteOnMainQueue(^{
            if (atomic_load(&cancelled)) {
                return;
            }
            UIImage *image = RCTImageFromLocalAssetURL(imageURL);
            if (image) {
                if (progressHandler) {
                    progressHandler(1, 1);
                }
                completionHandler(nil, image);
            } else {
                NSString *message = [NSString stringWithFormat:@"Could not find image %@", imageURL];
                RCTLogWarn(@"%@", message);
                completionHandler(RCTErrorWithMessage(message), nil);
            }
        });
        return ^{
            atomic_store(&cancelled, YES);
        };
    }
    NSString *cacheKey = [[SDWebImageManager sharedManager] cacheKeyForURL:imageURL];
    UIImage *image = [[SDWebImageManager sharedManager].imageCache imageFromCacheForKey:cacheKey];
    if (image) {
        if (progressHandler) {
            progressHandler(1,1);
        }
        completionHandler(nil,image);
    }
    [[SDWebImageDownloader sharedDownloader] downloadImageWithURL:imageURL options:SDWebImageDownloaderLowPriority progress:^(NSInteger receivedSize, NSInteger expectedSize, NSURL * _Nullable targetURL) {

    } completed:^(UIImage * _Nullable image, NSData * _Nullable data, NSError * _Nullable error, BOOL finished) {
        if (image) {
            dispatch_async(dispatch_get_global_queue(0, 0), ^{
                [[SDWebImageManager sharedManager].imageCache storeImage:image forKey:cacheKey completion:NULL];
            });
            RCTExecuteOnMainQueue(^{
                if (progressHandler) {
                    progressHandler(1,1);
                }
                completionHandler(nil,image);
            });
        } else {
            RCTLogWarn(@"%@", error.localizedFailureReason);
            completionHandler(error,nil);
        }
    }];
    return ^{
        atomic_store(&cancelled, YES);
    };
}

- (float)loaderPriority
{
    return 10.0f;
}

这里注意， 因为React也可以加载nativebundle的图片，所以在加载图片的时候需要判断图片来源是native的asset里面还是远程图片， 如果是远程URL，则替换成SDWebImage去加载。 至此，RN的图片加载缓存组件已经替换成SDWebImage， 和native打通，共用图片缓存逻辑，共同管理图片缓存。
