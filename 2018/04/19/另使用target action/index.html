另使用target action

一般而言，遵循MVC或者MVC衍生出的框架的设计要求所有的视图操作都放在V里面， 页面的一些点击事件的处理逻辑还需要通过C来协调。 iOS里面有提供诸如Target-Action,Delegate,Block等方式进行处理。 对一般只要一两个的时候可能更偏向于使用block来进行回掉。 如果一旦数量有点多的话，就有点不雅观了。比如像下图这样

这个时候，直接使用IMP去执行函数调用 可能看上去会更优雅一点。

首先在View中声明target和action

初始化View的时候进行赋值

在View中添加对应的点击事件

通过函数指针直接可以在target也就是VC中找到对应的函数实现了。

IMP

typedef id (*IMP)(id, SEL, ...);

IMP实际上就是一个函数指针，它指向函数实现的首地址。通过获取IMP，可以直接执行IMP指向的函数实现。

上述是直接通过函数指针

void (*function) (id, SEL, NSInteger) = (void*)imp;

找到对应的函数实现

function (self.target, self.action, type);
