痛苦的体验-CGAffineTransform

勤看苹果API真的真的很重要，记录一次比较痛苦的过程。

iOS之仿射变换

最近项目中有一个功能，难倒了同事，根据APP里提供的图片平移、缩放、旋转， 最后的位置信息保留给服务端生成SVG信息。 简而言之， 使用View的矩阵信息来保留图片的相对位置。

在iOS中，矩阵是用CGAffineTransform来描述信息的，

struct CGAffineTransform {
  CGFloat a, b, c, d;
  CGFloat tx, ty;
};
通常提供View的矩阵描述是二维矩阵， 
| a,  b,  0 |
| c,  d,  0 |
| tx, ty, 1 |
对UIImageView的平移、旋转、缩放分别使用 
CGAffineTransformTranslate
CGAffineTransformRotate
CGAffineTransformScale
来表示， 其中 CGAffineTransformRotate 会改变UIImageView的Bounds, 其真实描述的frame是 初始UIImageView 的外切矩形的frame， 然而通过查阅SVG的标准格式， SVG是以左上角为原点的相对布局方式。 如果我们仅仅通过Transform的叠加最后会发现 矩阵的缩放因子和旋转因子也就是 CGAffineTransform的a,b,c,d数据是正常的，tx,ty由于外切矩形的改变，导致最后的tx、ty是一个错误的值。 我是怎么进坑的呢？ 首先想到的当然是 初始化UIImageView的时候改变它的layer的锚点(anchorPoint)为左上角， 
_cmMaskView.layer.anchorPoint = CGPointZero;

花了半天计算，

CGFloat imgWidth =  CGImageGetWidth(self.currentMoudleImage.CGImage);
CGFloat imgHeight = CGImageGetHeight(self.currentMoudleImage.CGImage);
CGFloat ratio = MIN(self.editView.scrollView.frame.size.width / imgWidth, self.editView.scrollView.frame.size.height / imgHeight);
CGAffineTransform transform = self.editView.cmMaskView.transform;
transform.tx = (self.editView.cmMaskView.origin.x)/ratio;
transform.ty = (self.editView.cmMaskView.origin.y+maskOriginY)/ratio;

结果transform是对的， 但因为我们改变了锚点， 导致图片的平移、缩放、旋转都是针对锚点来展开的， 这样拿去给产品说 会被骂死的。

然后又想在手势的方法里面自己计算每次通过手势而改变的角度， 然后

简直算死了，最后花光了脑细胞发现还是不能用。。。。。。。。。。后来觉得就算算出来了这个办法简直太Low了。

然后细细想来，当时决定改变锚点位置，改变前后仅仅是CGAffineTransform的tx,ty不一样， 有没有什么办法求出在外切矩形的范围内求出 初始化时候的左上角呢？ 一步一步逼近答案。 翻遍了苹果提供API,果然是神器，

CGPointApplyAffineTransform

首先在获取仿射变换结束后还原没进行仿射变换的frame

- (CGRect)originalFrame
{
    CGAffineTransform currentTransform = self.editView.cmMaskView.transform;
    self.editView.cmMaskView.transform = CGAffineTransformIdentity;
    CGRect originalFrame = self.editView.cmMaskView.frame;
    self.editView.cmMaskView.transform = currentTransform;
    return originalFrame;
}

因为我们的最终目的是要求出UIImageView经过仿射变换之后的正确的左上角具体值，

- (CGPoint)newPointInView:(CGPoint)thePoint
{
    CGPoint offset = CGPointMake(thePoint.x - self.editView.cmMaskView.center.x, thePoint.y - self.editView.cmMaskView.center.y);
    CGPoint transformedPoint = CGPointApplyAffineTransform(offset, self.editView.cmMaskView.transform);
    return CGPointMake(transformedPoint.x + self.editView.cmMaskView.center.x, transformedPoint.y + self.editView.cmMaskView.center.y);
}

最后调用,获取我们最终想要的Transform

- (CGAffineTransform)fetchOriginTransform
{
    CGFloat imgWidth =  CGImageGetWidth(self.currentMoudleImage.CGImage);
    CGFloat imgHeight = CGImageGetHeight(self.currentMoudleImage.CGImage);
    CGFloat ratio = MIN(self.editView.scrollView.frame.size.width / imgWidth, self.editView.scrollView.frame.size.height / imgHeight);
    CGPoint point = [self newPointInView:[self originalFrame].origin];
    CGAffineTransform transform = self.editView.cmMaskView.transform;
    transform.tx = point.x/ratio;
    transform.ty = point.y/ratio;
    return transform;
}

这里重点记录一下CGPointApplyAffineTransform

Returns the point resulting from an affine transformation of an existing point.

返回一个已经存在的点仿射变换之后所产生的点。 最终我们通过变换矩阵的左上角来确保CGAffineTransform的tx,ty的正确， 最后保证CGAffineTransform的正确来使用标准的SVG格式。
