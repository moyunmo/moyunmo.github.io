深入理解Block

经常写block只知道对象在block内部是不能直接进行赋值操作的， 如果需要修改、赋值需要在前面添加__block修饰符修饰，可是，为什么呢？

不用__block为什么不能在Block内部修改Block外的局部变量

比如、声明一个Block

int main(int argc, char * argv[]) {
    int val = 1;
    void (^blk) (void) = ^{
        NSLog(@"%d",val);
    };
    blk();
    return 0;
}

可以看到Block其实是一个*__main_block_impl_0*的结构体，其中 *val* 成为结构体中的成员，结构体的成员变量类型和自动变量类型安全相同，并且Block语法表达式中没有使用的自动变量不会被追加。还有其他成员，比如*__block_impl*，也是如下的这样一个结构体 
struct __block_impl {
  void *isa;
  int Flags;
  int Reserved;
  void *FuncPtr;
};

然后在执行Block的时候怎么去访问val？ Block执行的时候，其实是执行 __main_block_func_0，参数是struct __main_block_impl_0 *__cself，这个cself相当于C++中的this、OC中的self，然后通过__cself->val访问。其实理论上说可以通过指针来修改val这个变量，但是main函数中的val和__main_block_func_0其实不在同一个作用域，__main_block_func_0执行时，变量还在栈中，但是在很多情况下，Block是作为参数传递以供后续回调的，如果在这种情况下，Block执行时，局部变量已经被释放了，在用指针指向的时候肯定就出错了， 所以局部变量不允许修改是合理的。

什么情况下允许Block改写值？

  • 全局变量
  • 静态全局变量
  • 静态变量

分别添加一个全局变量、静态全局变量、静态变量，

int global_val = 1;
static int static_global_val = 2;
int main(int argc, char * argv[]) {
    int val = 1;
    static int static_val = 3;
    void (^blk) (void) = ^{
        NSLog(@"%d",val);
        global_val *= 2;
        static_global_val *= 3;
        static_val *= 4;
    };
    blk();
    return 0;
}

为什么__block修饰符修饰的可以在Block中修改

block说明符（block storage-class-specifier）类似于static、auto说明符，它们用于指定变量值设置到哪个存储域，比如auto标识作为自动变量存储在栈区，static标识作为静态变量存储在.data区。
在main函数中将val的定义前加上__block


int main(int argc, char * argv[]) {
    __block int val = 1;
    void (^blk) (void) = ^{
        val = 2;
    };
    blk();
    return 0;
}

Block存储域

Block其实是Block的结构体类型的自动变量，block是block变量的结构体类型的自动变量， 为什么Block_byref_val_0中有个指向自己实例的指针变量呢？ 在前面第二段中有提到全局变量和block可以在Block内部修改， 但是一般情况下，如果出栈了在用cSelf或者__forwarding去指向已经出栈的变量，为什么不会出错呢？

Block提供了将Block和block变量从栈复制到堆的方法来解决这个问题，而栈上的forwarding在复制的时候，指向的其实是堆上的实例，这样就保证访问的过程中不会出错。 在ARC下，这些情况会把栈上的Block复制到堆：

  • 调用copy或者__Block_copy
  • Block作为函数返回值返回时
  • 将Block赋值给附有__strong修饰符id类型的类或Block类型成员变量时
  • 在Cocoa框架中包含usingBlock或GCD的API传递Block时
