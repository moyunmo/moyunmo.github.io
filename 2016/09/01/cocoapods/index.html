使用CocoaPods创建公有库

发表于 2016-09-01

本文只做个人记录使用。

1,确保工作区关联一个git, 比如公有库在github上建一个仓库地址（https://github.com/moyunmo/CMDemo.git）

2,创建一个全新的工程，使用命令

pod lib create 你想要的pod名
然后会弹出一系列的选项 3,cd到当前工程目录下，关联git 
git remote add origin https://github.com/moyunmo/CMDemo.git

4,往CMDemo/Classes下添加源代码

5,修改CMDemo.podspec配置文件

Pod::Spec.new do |s|
  s.name             = 'CMDemo' #名称
  s.version          = '0.0.1' #版本号
  s.summary          = 'Test demo' #简短介绍

#下面是详细介绍
  s.description      = 'Test demo,Test demo,Test demo,Test demo'

  s.homepage         = 'https://github.com/moyunmo/CMDemo' #主页,这里要填写可以访问到的地址，不然验证不通过
  # s.screenshots     = 'www.example.com/screenshots_1', 'www.example.com/screenshots_2'  #截图
  s.license          = { :type => 'MIT', :file => 'LICENSE' } #开源协议
  s.author           = { 'Moyun' => 'moyunmo@hotmail.com' } #作者信息
  s.source           = { :git => 'https://github.com/moyunmo/CMDemo', :tag => '0.0.1' } #项目地址，这里不支持ssh的地址，验证不通过，只支持HTTP和HTTPS，最好使用HTTPS

  s.ios.deployment_target = '8.0' #支持的平台及版本

  s.source_files = 'CMDemo/Classes/**/*'   #代码源文件地址，**/*表示Classes目录及其子目录下所有文件，如果有多个目录下则用逗号分开，如果需要在项目中分组显示，这里也要做相应的设置

  # s.resource_bundles = {
  #   'CMDemo' => ['CMDemo/Assets/*.png']
  # }    #资源文件地址

  # s.public_header_files = 'Pod/Classes/**/*.h' #公开头文件地址
  # s.frameworks = 'UIKit', 'MapKit' #所需的framework，多个用逗号隔开
  # s.dependency 'AFNetworking', '~> 2.3' #依赖关系，该项目所依赖的其他库，如果有多个需要填写多个s.dependency
end

6,提交源码，git tag和pod版本一致

$ git add .
$ git commit -m "init commit"
$ git push origin master
$ git tag -m 'init' '0.0.1'
$ git push --tags

7,推送podSpec

pod trunk me ，检测是否注册，如果没有,使用pod trunk register，之后

$pod trunk push [NAME.podspec] #部署公共的Podspec,后面加--allow-warnings 允许警告

8,使用Pod

$ pod 'CMDemo', '~> 0.0.1'

9,更新维护

升级代码 -> 根据具体改动，编辑name.podspec -> pod lib lint验证 -> 将项目推送到远端仓库,打上新的tag -> 提交到Spec Repo中，

$pod trunk push [NAME.podspec]

二、其他常用操作

  • 指定私有源
    source 'https://github.com/CMSpecs/Specs.git'  
source 'https://github.com/CocoaPods/Specs.git'
  • 指定tag
    pod 'Cmall_Base_Utils',  :git => 'http://git.cmall.com/ios/Cmall_Base.git', :tag => '0.0.1'
  • 指定commit
    pod 'Cmall_Base_Utils',  :git => 'http://git.cmall.com/ios/Cmall_Base.git', :commit => '1a31b864d1d56b1aaed0816c10bb55cf2e078bb8'
  • 使用子仓库
    pod 'CmallSDK/CmallSDK_Base'
    或者
    pod 'CmallSDK', :subspecs => ['CmallSDK_Base', 'CmallSDK_UI']
  • 指定依赖库的配置文件
    pod 'PonyDebugger', :configuration => ['Release']
  • 输出日志
    pod install --verbose
  • install/update提速、不更新本地索引
    pod install --no-repo-update  
pod update --no-repo-update
  • pod中含有category，自动给工程添加-ObjC,在podSpec中添加
    s.xcconfig = { "OTHER_LINK_FLAG" => '$(inherited) -ObjC'}
  • 项目编译遇到第一个错误的时候就停止
    在后面添加 --fail-fast

三 、 使用问题

1，[!] Unable to find a pod with name, author, summary, or descriptionmatching CMDemo

可能原因，pod setup没执行成功。1）gem版本太低。2）github访问不了。3）.cocoapods检索文件有问题

1）更新cocoapods。

$ sudo gem update --system

2）检测github链接

$ ping github.com

3）rm ~/Library/Caches/CocoaPods/search_index.json,移除索引，重新执行

pod search

参考链接：http://blog.wtlucky.com/blog/2015/02/26/create-private-podspec/
